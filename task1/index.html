Building Operating System

Author: Fatima Naeem

Building 64-bit Operating System Kernel from scratch.

Tools used

  • Visual Studio Code for writing and compiling the code

  • Docker for building the Operating System

  • Qemu for emulating the OS

Building Image

Using gcc compilation tools available on docker hub, environment is built and then a Docker Image is built from Windows against a non-Windows docker host and then an instance of that image is spinned up.

Essential files

  • header.asm

    It consists of some data to be included in the operating system binary.

    This magic data is important so that bootloaders can understand that we have an operating system that can be run on this computer.

    Bootloader’s job is to locate the operating system and start it up. Multi-boot 2 specification is used for this OS, which is supported by most bootloaders.

  • main.asm

    This file is the entry point into the OS.

  • linker.ld

    It describes how to link the operating system together.

    The entry point and all the individual sections, such as the boot and the text sections, are specified.

  • grub.cgf

  • Makefile

    This organizes all the build commands.

    It also ensures only the modified files are rebuilt to make the process fast.

    It contains variables to hold all the assembly source files and assembly object files.

    Makefile contains the commands to build the object files from the corresponding source file when it is modified. Nasm compiles the assembly source files.

  • kernel.iso

    This is the final iso file. This is generated when the instructions for building the operating system are executed.

This is the ouput after emulating the OS on Qemu
